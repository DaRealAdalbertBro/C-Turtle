Developed by Jesse Walker as part of a 2019 summer internship at Berea College!

C-Turtle is an attempt to provide Pythons (and thus Logos) Turtle Facility to C++!

As an aspiring future Computer Science educator, I have noticed the value of having visible, tangible output beyond lines of text while in the early stages of learning basic, related, concepts. While hesitant to learn Python at first (it definitely was not my first language), I quickly fell in love with the value behind what Turtles have to offer to a learning environment. It lets students explore Computer Science in familiar territory (Cartesian Planes are taught at a young age), while having something to observe which visually walks them through all of their function calls.

With this in mind, I noticed many of my peers struggling to grapple the syntax of C++ after having experience with Python. "How could this be?", I thought to myself; "The core concepts are largely the same, what could I do to help them?". I later tried introducing the syntax of C++ to my assignment partners under terms that would be familiar to them– which quickly became the inspiration for this project. My peers seemed to learn meaningful information quicker when they have something familiar to work with.

Usage Of & Building This Project

This project contains a makefile to build a statically linked library. That said, in a learning environment, I encourage instructors and students to merely copy and paste the source into their projects, to simplify the process.

Under VisualC++ (MSVC) and Windows, the required system libraries are automatically linked (kernel32 & gdi32).

Under Unix-like environments (Clang, GNU/MSYS/MinGW/Cygwin, LLVM), you must link against pthread and X11. You can install the development libraries for these by running the following commands in a terminal, provided you are operating under a linux distribution and Synaptic is the preferred package manager:

sudo apt-get install libpthread-stubs0-dev
sudo apt-get install libx11-dev

Example Comparison

Below is a comparison of two well-formed examples, showing how to draw a simple purple square in both Python (left) and C-Turtle (right).

Python C-Turtle

import turtle
turt = turtle.Turtle()
turt.fillcolor("purple")
turt.speed("slowest")
turt.begin_fill()
for i in range(4):
turt.forward(50)
turt.right(90)
turt.end_fill()
turt.bye()

#include "CTurtle.hpp"
namespace ct = cturtle;
int main(int argc, char** argv) {
ct::TurtleScreen scr;
ct::RawTurtle rt(scr);
rt.speed(ct::TS_SLOWEST);
rt.fillcolor(ct::Color::purple);
rt.begin_fill();
for(int i = 0; i < 4; i++){
rt.forward(50);
rt.right(90); }
rt.end_fill();
scr.bye();
return 0;
}

The output of both of these programs are nearly identical!
