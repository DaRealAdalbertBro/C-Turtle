1 ## Setup
2 ### 1.) Getting CMake
3 C-Turtle uses an Open-Source tool called CMake to generate "make files", or in the case of Visual Studio, solutions.
4 You can get CMake [at this link here](https://cmake.org/download/). You can download the correct version for your
5 computer by going to the *Binary Distrubitions* section, where you will find cross-platform versions for Windows, Mac, and Linux.
6 From here, you need to merely install it. All default options are perfectly fine.
7 If you are operating on Linux, scroll past the following image to make this process somewhat easier for yourself.
8 
9 ![Image of CMake Website][cmake_web_bindist]
10 
11 #### Side-Note for Linux Users
12 CMake is frequently found and installed through the use of package managers. Furthermore, you must install CMake-QT-Gui, the graphical interface for the CMake tool.
13 This can be done on Ubuntu-based distributions through the following command. The graphical tool is installed by default on the Windows distribution of CMake.
14 
15 ```shell
16 sudo apt-get install cmake && sudo apt-get install cmake-qt-gui
17 ```
18 
19 ### 2.) Download the C-Turtle Repo
20 Next, it is necessary to download the C-Turtle code itself. This can be done by visiting the [GitHub Repository](https://github.com/walkerje/C-Turtle)
21 where the code is located, and downloading it. To do so, make your way to the green *"Clone or Download"* button at the top right, then choose to download the ZIP distribution. The steps are highlighted in blue in the following image. Then extract the downloaded ZIP to a memorable location.
22 
23 ![Image of C-Turtle Repo][cturtle_repo_dl]

26 ### 3.) Configure CMake to build C-Turtle
27 Configuring CMake is quick and easy thanks to its graphical setup. Open the recently installed CMake tool (May be named CMake-Gui),
28 and you will be greeted with the following screen:
29 
30 ![Image of CMake Program][cmakegui_client_img]
31 
32 The notable components of this interface are the text-boxes labeled "Where is the source code" and "Where to build the binaries",
33 and the buttons labeled *Configure* and *Generate*. For the box labeled *Where is the source code:*, browse to where you have downloaded and extracted the C-Turtle repo. This folder should contain files such as *sandbox.cpp* and *CMakeLists.txt*, as well as a folder labeled *src* in which the source code of C-Turtle is located.
34 
35 In the text box labeled "Where to build the binaries", any empty folder will do. For the sake of example, we will set this to a directory named "build" which
36 will be located in the same directory as where the source files are. CMake may ask you if it can create a destination folder, if it doesn't already exist. Click "Yes".
37 
38 ![Image of CMake Setup][cmakegui_setup_img]
39 
40 Next, click the button labeled "Configure" located towards the bottom left of the CMake client. This should bring up a dialog box asking what to generate "make files" or "solutions" for. Select whatever toolchain you would prefer; on Linux, you likely want Unix makefiles, however on Windows you likely want to create Visual Studio solutions.
41 Select whichever option applies for you and click the button labeled "Finish".
42 
43 ![Image of CMake Selection][cmakegui_choose_img]
44 
45 This action makes CMake process the C-Turtle project for the target you selected (observable by the textual output at the bottom of the client).
46 When this is complete, various options will appear, highlighted in red. The red is not an indication of an error, but rather an indication that the options
47 are new (e.g, were not present prior to clicking "Configure").
48 
49 ![Image of CMake Build Selection][cmakegui_almostdone_img]
50 
51 From here, we can choose what we would like to have generated. For the sake of example, we shall generate the "Sandbox" build, which will generate a "Make File" (or Solution, in Visual Studio) ready-made for students to use in a learning environment. You may also choose to build the Examples, which will add source code of the Examples
52 to the Makefile or Visual Studio solution.
53 
54 After this, navigate to the directory previously chosen for the text box "Where to build the binaries". Depending on the target chosen for your use case (UNIX Makefiles or Visual Studio) there will either be a Makefile ready to use, or a Visual Studio solution file. From here you can edit C-Turtle's source, or open the solution, which marks
55 the completion of C-Turtle's setup.
56 
57 #### Side-note for individuals not using Visual Studio
58 
59 The Sandbox build draws its source from the source file called "sandbox.cpp", located in the root directory of the repo. As such, whenever you would like to
60 mess around with the Sandbox environment, simply edit the contents of this aforementioned source file and run the Makefile generated previously.
